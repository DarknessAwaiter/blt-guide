# Мини-разметка для текстов BLT

Короткие теги, которые можно использовать во всех `data/*.js`, чтобы единообразно выделять команды, цену и важные фрагменты. Обработка тегов реализована в `scripts.js` в функциях `renderRichText`, `formatGold` и `stripMiniTags`.

---

## Кратко по синтаксису

Используются три коротких тега:

- `$c(...)` — команда чата или пример команды  
- `$p(...)` — стоимость в золоте (число → форматируется и добавляется `⦷`)  
- `$k(...)` — важный фрагмент / ключевая фраза  

Все теги имеют вид: `$x(любой_текст_до_закрывающей_скобки)`.

---

## Где можно использовать теги

Теги работают во всех текстовых полях, которые проходят через `renderRichText(raw)`:

- `data/commands.js`  
  - `description`  
  - `cost`  
  - `syntax`  
  - `example`  
  - `subcommands[].desc`  
  - `subcommands[].cost`  

- `data/rewards.js`  
  - `description`  
  - `command`  
  - (`effect`, если решим его отображать)  

- `data/intro.js`  
  - `steps[].text`  

- `data/achievements.js`  
  - `requirement`  
  - `reward`  

- (опционально) `data/classes.js`  
  - `levels[].effect`  
  - `levels[].req`  

Любой текст, который не проходит через `renderRichText`, теги не понимает и покажет сырую строку.

---

## Поведение каждого тега

### `$c(...)` — команда чата

Используется для подсветки команд и примеров использования.

В данных:

```js
description: "Используй $c(!усыновить), чтобы создать героя.",

text: "Команда $c(!бой) призывает героя на сторону стримера.",

syntax: "$c(!фокус <название_фокуса>)",
example: "$c(!фокус атлетичность)"
```

На странице:

- фрагменты внутри `$c(...)` отображаются моноширинным шрифтом и акцентным цветом (`.tag-cmd` в CSS).

---

### `$p(...)` — стоимость в золоте

Используется для числовых значений стоимости. Внутри **только число**, без символа `⦷` и без форматирования тысяч.

В данных:

```js
description: "Стартовое золото: $p(50000).",

cost: "1=$p(25000)<br>2=$p(50000)<br>3=$p(100000)",

reward: "$p(100000) + титул"
```

Обработка:

- берутся только цифры из строки внутри скобок;  
- число форматируется через `toLocaleString("ru-RU")`;  
- добавляется символ `⦷`.

На странице:

- `$p(50000)` → `50 000⦷`  
- `$p(1000000)` → `1 000 000⦷`  
- визуально оформляется через класс `.tag-cost`.

---

### `$k(...)` — ключевой фрагмент / важная фраза

Используется для подчёркивания важных частей текста (условия, предупреждения, советы).

В данных:

```js
description: "$k(Важно:) использовать команду только вне боя.",

text: [
  "Команда $c(!бой) призывает героя на сторону стримера.",
  "$k(Совет:) сначала прокачай снаряжение, потом свиту."
].join("\n")
```

На странице:

- текст внутри `$k(...)` отображается приглушенным цветом и курсивом (`.tag-kw` в CSS), чтобы визуально отличаться, но не спорить с командами и ценой.

---

## Как это реализовано в scripts.js

### Функция `formatGold(numStr)`

```js
function formatGold(numStr) {
  const n = Number(String(numStr).replace(/\D/g, ""));
  if (!Number.isFinite(n)) return numStr;
  return n.toLocaleString("ru-RU") + "⦷";
}
```

- Удаляет все не цифры из `numStr`.  
- Преобразует в число и форматирует по русской локализации.  
- Добавляет символ `⦷`.

### Функция `renderRichText(raw)`

```js
function renderRichText(raw) {
  if (!raw) return "";
  let s = String(raw).replace(/\n/g, "<br>");

  s = s.replace(/\$c\((.+?)\)/g, '<span class="tag-cmd">$1</span>');
  s = s.replace(/\$p\((.+?)\)/g, (_, num) => {
    return '<span class="tag-cost">' + formatGold(num) + "</span>";
  });
  s = s.replace(/\$k\((.+?)\)/g, '<span class="tag-kw">$1</span>');

  return s;
}
```

- Меняет переводы строк на `<br>`.  
- `$c(text)` → `<span class="tag-cmd">text</span>`  
- `$p(num)` → `<span class="tag-cost">formatGold(num)</span>`  
- `$k(text)` → `<span class="tag-kw">text</span>`

### Функция `stripMiniTags(text)`

Используется перед копированием команд в буфер обмена:

```js
function stripMiniTags(text) {
  if (!text) return "";
  return String(text)
    .replace(/\$[cpk]\((.+?)\)/g, "$1")
    .replace(/<[^>]+>/g, "");
}
```

- Удаляет `$c/$p/$k`, оставляя только содержимое в скобках.  
- Удаляет возможные HTML‑теги.  
- Благодаря этому в буфер попадает «чистая» команда без подсветки и символов форматирования.

---

## Рекомендации по использованию

- Не вкладывать теги друг в друга (например, `$c($p(50000))`) — поведение не определено.  
- Внутри `$p(...)` использовать только цифры, форматирование и символ `⦷` делает фронт.  
- Для команд всегда использовать `$c(...)`, особенно в `syntax` и `example` — так легче визуально выцепить нужный фрагмент.  
- `$k(...)` использовать для «Важно:», «Требуется:», «Совет:» и похожих пометок.

## Примеры «до/после»

### Команда с ценой и подсветкой

```js
// data/commands.js
{
  name: "!фокус",
  category: "hero",
  description: [
    "Команда $c(!фокус) покупает очки фокуса в навыки.",
    "$k(Совет:) сначала качай фокус в оружие, которым чаще всего пользуешься."
  ].join("\n"),
  syntax: "$c(!фокус <название_навыка>)",
  example: "$c(!фокус атлетичность)",
  cost: "1=$p(30000)<br>2=$p(40000)<br>3=$p(50000)<br>4=$p(60000)<br>5=$p(75000)"
}
```

На странице:

- команды и синтаксис выделены,  
- цена красиво форматируется и отображается с `⦷` под описанием,  
- советы аккуратно подсвечены как второстепенная, но важная информация.
